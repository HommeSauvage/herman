## Setup
- Turn this starter into a simple marketing website focused entirely on frontend design and copy. Remove Restate, auth, campaigns, todos, admin dashboard, database, and any other backend or server-side functionality.
- Build this as a static site. Do not use SSR, server functions, API routes, or database access. All content is hardcoded no dynamic server rendering.
- Remove any auth/database/admin/restate code to keep it light. Do the same for the packages.
- Structure the page with clear sections: hero, features or benefits, social proof, and a call to action. Mimic the layout and rhythm of polished SaaS landing pages (Linear, Vercel, or Stripe marketing pages are good references).
- Make the page fast and lightweight
- Add SEO basics in static markup: page title, meta description, and Open Graph tags for link previews.
- For CTAs, use external links, `mailto:` links, or anchor buttons — do not build forms that require a server or database.

## Questions
- Ask the user who the product or idea is for.
- Ask them what main benefit or problem they want the hero to highlight.
- Ask about the call to action: a link to an external URL, a `mailto:`, whatsapp link, contact link, or a scroll-to-section button.
- Ask whether they want social proof: customer testimonials, logo bar, stats/numbers, or none for now.

## Guidance
This is a static marketing landing page. Focus on design, typography, layout, and copy.
Do not add admin panels, auth, databases, server functions, or SSR unless the user explicitly asks.
